Chapter 1 · Introduction

AI Agents — my notes 🤖

From An Illustrated Guide to AI Agents — Jay Alammar & Maarten Grootendorst (O'Reilly, Early Release)

The one-line version: An AI Agent is an LLM (the "brain") that has been given eyes, hands, and a memory so it can think, decide, and actually do things in the world — not just chat.
Read me first You don't need a tech background for this. Everything below is an everyday analogy first, the "proper" word second. If you only remember one picture, remember the brain + eyes + hands + notebook one. That's an agent.

1 Why is everyone suddenly talking about this?

  • 2022 — ChatGPT arrives. It could talk. Huge deal, but it only produced text.
  • 2024 — AI Agents arrive. The big leap: they can act with autonomy, chase a goal, and interact with the world — write whole codebases, remember long conversations, reason step by step.
  • The key new ingredient is agency — the AI decides what to do, when, and how, instead of being spoon-fed every step.
  • They're already showing up in coding, finance, healthcare, marketing & research.
🧑‍💼
Analogy: A plain LLM is like a brilliant person locked in an empty room who can only talk. An agent is that same person given a phone, a computer, and a to-do list — now they can actually get things done.

2 So… what is an AI Agent?

The book starts from a classic 60-year-old definition of any agent (from the famous AI textbook by Russell & Norvig):

DEFINITION

"An agent is anything that can perceive its environment through sensors and act on it through actuators."

Break that into four simple parts — this is the skeleton of the whole book:

  • Environment = the world the agent lives in & works on (incl. you, the user!)
  • Sensors = how it observes the world (its eyes & ears)
  • Actuators = how it acts on the world (its hands = tools)
  • Effector = the "brain" that decides: observation → action

Now map it to a real AI agent

Those textbook words become very concrete once you plug in an LLM:

  • Effector / brain → a "reasoning" LLM (a model that can think in steps)
  • Actuators / hands → the LLM's tools (search, calculator, code, APIs…)
  • Sensors / eyes → the LLM reading images or sound, not just text
  • Environment → the digital world + you. No user prompt = the agent just sits there.
🎙️
Trainer tip: In a room, draw the little robot: brain in the head, eyes, hands, and a notebook in its pocket. Every concept in this chapter clips onto one of those four body parts. People remember bodies, not bullet points.

3 The brain, part 1: the LLM (Ch 2)

Strip away everything fancy and an LLM (Large Language Model) does exactly one thing:

CORE IDEA

It predicts the next word (well, the next token) based on the text so far. That's it.

  • Tokens = little chunks of words. Splitting into chunks lets the model handle words it's never seen.
  • It guesses the next token, adds it to the text, then guesses again… over and over.
  • That repeat-loop has a name: autoregression. It's how one guess becomes a full paragraph.
📱
Analogy: It's your phone's autocomplete — but trained on a huge chunk of the internet, so its "next word" guesses are shockingly good.

4 The brain, part 2: Reasoning LLMs (Ch 3)

  • For years we made models better by making them bigger — more data, more compute. This is train-time scaling.
  • But bigger got hugely expensive for smaller and smaller gains — a ceiling.
  • New trick, copied from humans: make the model "think out loud" before answering.
  • The model spends extra effort writing out its steps first → much better at hard, multi-step problems. The steps are usually hidden; you just get the summary.
  • This is what lets agents plan, pick tools, and fix their own mistakes — so reasoning LLMs are the star of the book.
  • Trade-off: reasoning is slower & pricier. For quick, cheap answers, a normal LLM is fine.
✍️
Analogy: A maths student who shows their working gets the hard questions right far more often than one who blurts the first number that comes to mind.
Chapters 2 + 3 = "the brain" Ch 2 is the plain LLM. Ch 3 is the reasoning LLM. Together they're the brain of every agent. Everything after this is about giving that brain a body.

5 Giving the brain a body: augmenting the LLM (Ch 4–6)

A brilliant brain alone can't remember you, can't press buttons, and can't make a plan. We bolt on three add-ons to fix that. 👇

Memory (Ch 4)

  • An LLM by itself is stateless — it forgets everything the second it replies. Great for one question, useless for a real conversation.
  • Simple fix: paste the earlier conversation back into the next prompt so it "remembers."
  • Real memory is richer — short-term vs long-term, just like ours.
  • But too much info = information overload → worse decisions. Choosing what to keep in front of the model is called context engineering.
🐠
Analogy: Without memory the model is a goldfish — new question, blank slate. Memory is the notebook it carries so it can pick up where you left off.

Tools (Ch 5)

  • Tools let the LLM reach out and do things: search the web, run code, use a calculator, hit an API.
  • Catch: the LLM can't press the button itself. It's text-in / text-out — all it can do is write down the intent.
  • Example: ask "What is 2.3 × 8.1?" and it outputs the text multiply(2.3, 8.1). Our code sees that, actually runs it, and feeds the answer back.

Planning & reflection (Ch 6)

  • Planning = break a big task into small do-able steps (task decomposition) and do them one at a time.
  • Reflection = after each step, look back: did that work? what next? should I change the plan?
  • Together they form a loop: plan → act → reflect → adjust → repeat. This is what makes agents feel human — they catch and fix their own mistakes.
PUT IT TOGETHER

Reasoning LLM + memory + tools + planning/reflection = an AI Agent. Anthropic calls the tool-and-memory version the "Augmented LLM."

6 Setting it loose: the agentic system (Ch 7)

Autonomy is a dial, not a switch

  • Agents can have a little freedom or a lot. It's a spectrum.
  • Full autonomy is powerful but risky, so we add guardrails to stop destructive actions.
  • The book's take: if it shows goal-directed behaviour and makes decisions, it counts as an agent — even a partly-supervised one.

Where agents genuinely shine

  • Coding 💻 — the #1 use today. Clear goal, some freedom in how. Writes, runs & tests code.
  • Deep research 🔬 — give a topic, it hunts sources (ArXiv, PubMed, Scholar) on its own.
  • Automation ⚙️ — tidying & standardising messy data across systems (e.g. hospital records).
The sweet spot Agents are best when the goal is clear but the path isn't — open-ended problems where you can't script every step in advance.

Using them responsibly ⚠️

  • Human in the loop — a person checks important decisions before they go through.
  • Guardrails — don't build a fully-autonomous agent when a limited one is safer & just as good.
  • Misinformation / hallucination — agents are still LLMs; they can be confidently wrong. Add checks where correctness is critical.
🎙️
Trainer tip — the killer line: "You'd happily let an agent draft your code. You would not let it diagnose a patient with no human checking." It's all about context. This one sentence usually lands the whole ethics point.

Evaluating agents is hard

  • Judging a chatbot = judge the text. Judging an agent = judge a whole sequence of actions.
  • You measure the outcome (did the task get done?) and the process (efficiently? safely? did it follow the rules?).
  • So evaluating an agent means evaluating an entire system, not just a model.

7 Special kinds of agents (Ch 8–13)

Part 2 of the book takes the basic agent and specialises it four ways:

  • Multi-agent teams 👥 (Ch 8) — several agents, each a specialist, often led by a supervisor agent (the "manager", running the smartest model) that splits up and hands out the work.
  • Multi-modal agents 🖼️🔊 (Ch 9–10) — can see images and hear/speak, not just read text. Needed for things like judging a website's design or talking hands-free.
  • Coding agents 💻 (Ch 11–12) — can actually run programs, read a codebase, write functions, fix bugs and test — an autonomous (or assistant) developer.
  • Tiny agents 🐜 (Ch 13) — smaller, cheaper, faster. Agents burn tokens unpredictably, so making them efficient (smaller models, cheaper inference) matters a lot.
✦ the whole chapter on one line ✦
Recap: Start with an LLM that predicts words → make it reason (think out loud) → give it memory + tools + planning → let it act with some autonomy (plus guardrails) → that's an agent. Then specialise it into teams, multi-modal, coding & tiny agents.

★ Interview cheat-sheet 🎤

Likely questions + short, confident answers you can say out loud. Tap to reveal.

Q. In one sentence — what's an AI agent?
An AI system that uses an LLM as its "brain," plus memory, tools and planning, to decide and take actions toward a goal — not just generate text.
Q. How is an agent different from ChatGPT?
ChatGPT talks; an agent acts. The agent adds autonomy — it chooses what to do, when and how, and can use tools and remember context to actually get a task done.
Q. What are the core parts?
Four: Environment (the world + user), Sensors (how it observes), Actuators (tools it acts with), and the Effector/brain (the reasoning LLM that decides). Or plainly: brain, eyes, hands, memory.
Q. Why do agents need "reasoning" LLMs?
Because planning, choosing tools and fixing mistakes are multi-step problems. A model that "thinks out loud" first solves those far better — like showing your working in maths.
Q. If an LLM can't press buttons, how does it use tools?
It only outputs text describing the intent (e.g. multiply(2.3, 8.1)). Surrounding software reads that, runs the real tool, and feeds the result back.
Q. What are the main risks / how do we stay safe?
They can act in the real world and can be confidently wrong (hallucinate). Safeguards: keep a human in the loop, add guardrails, and verify outputs where correctness is critical. Autonomy should match the stakes.
Q. Where are agents actually useful?
Best when the goal is clear but the path isn't: coding assistants, deep research, and automating messy, multi-step data work.
